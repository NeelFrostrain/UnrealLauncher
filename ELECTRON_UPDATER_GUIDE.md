## How It Works

### Auto-Update Flow

1. **App Startup** → `setupAutoUpdaterEvents(getMainWindow)` called
2. **Check GitHub** → Automatically fetches latest release from GitHub
3. **Compare Versions** → Uses `compareVersions()` to detect updates
4. **User Prompt** → Dialog asks to download update
5. **Background Download** → Installer downloads silently
6. **Update Ready** → Dialog prompts to restart and install
7. **Auto Install** → On restart or quit, update is installed

### Main Process (`src/main/updater.ts`)

```typescript
// Automatic events:
setupAutoUpdaterEvents(getMainWindow)

// Handles:
- update-available → Shows download dialog
- update-downloaded → Shows restart dialog
- download-progress → Sends progress to renderer
- error → Shows error message
```

### IPC Handlers Available

```typescript
window.electronAPI.checkForUpdates() // Manual check
window.electronAPI.downloadUpdate() // Download update
window.electronAPI.installUpdate() // Install & restart
window.electronAPI.getAppVersion() // Get current version
window.electronAPI.checkGithubVersion() // GitHub version info
window.electronAPI.onDownloadProgress() // Listen to progress
```

## Configuration

### 1. Update `electron-builder.yml`

Already configured for auto-updates. Key settings:

```yaml
appId: com.unreallauncher.app
productName: Unreal Launcher
directories:
  buildResources: build
  output: dist
files:
  - 'out/**'
  - 'resources/**'

# Windows installer
win:
  target:
    - nsis
  executableName: unreallauncher

# Publish to GitHub (optional, for auto-updates)
# Uncomment if using GitHub releases for updates:
# publish:
#   - provider: github
#     owner: NeelFrostrain
#     repo: UnrealLauncher
```

### 2. GitHub Release Setup

For **auto-updates to work**, you need:

1. **Create Release Tag** with version matching your app:

   ```
   Tag: v2.0.5
   Release Name: Version 2.0.5
   ```

2. **Upload Artifacts:**
   - `unreal-launcher-2.0.5-setup.exe` (Windows installer)
   - `latest.yml` (auto-generated by electron-builder)

3. **electron-builder creates `latest.yml`:**
   ```yaml
   version: 2.0.5
   files:
     - url: unreal-launcher-2.0.5-setup.exe
       sha512: <hash>
       size: 123456789
   path: unreal-launcher-2.0.5-setup.exe
   sha512: <hash>
   releaseDate: '2026-04-19T12:00:00.000Z'
   ```

### 3. Build & Release Steps

```bash
# 1. Increment version in package.json
npm version patch  # or minor, major

# 2. Build the app
npm run build:win

# 3. Create GitHub release
# - Go to: https://github.com/NeelFrostrain/UnrealLauncher/releases/new
# - Tag: v<version>
# - Upload files from dist/

# 4. Publish release
# - The app will automatically detect the update on next startup
```

## Advanced Usage

### Checking for Updates Manually

```typescript

const result = await window.electronAPI.checkForUpdates()
// {
//   success: true,
```

### Displaying Update Status

```typescript
// Listen to download progress
window.electronAPI.onDownloadProgress((progress) => {
  console.log(`Downloaded: ${progress.percent}%`)
  console.log(`Speed: ${progress.bytesPerSecond} B/s`)
  console.log(`${progress.transferred} / ${progress.total} bytes`)
})

// Download update manually
await window.electronAPI.downloadUpdate()

// Install update
await window.electronAPI.installUpdate()
```

## Troubleshooting

### Updates Not Detected

1. **Check GitHub Release:**
   - Tag must be: `v<version>` (with v prefix)
   - Must match version in `package.json`

2. **Verify `latest.yml` exists:**
   - Download from release assets
   - Should contain file hashes

3. **Enable Debug Logging:**
   ```typescript
   // In src/main/updater.ts, add:
   autoUpdater.logger = require('electron-log')
   autoUpdater.logger.transports.file.level = 'debug'
   ```

### Windows Defender Warning

Windows may block unsigned executables. To code-sign:

1. Get an EV Code Signing Certificate
2. Update `electron-builder.yml`:
   ```yaml
   win:
     certificateFile: path/to/cert.pfx
     certificatePassword: password
   ```

### Failed Downloads

- Check internet connection
- GitHub API rate limits (60 requests/hour unauthenticated)
- Repository is public and accessible
